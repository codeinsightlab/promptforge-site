下一条消息示例

SQL：select * from user where phone = 13800138000。

表字段：phone varchar(20)，有索引 idx_phone(phone)。EXPLAIN 显示 type=ALL，key=null。

请帮我判断为什么索引没生效。

不用完全照抄，按你手头已有的信息发送即可；材料不全时，GPT 会继续追问。

输出效果预览

原因：phone 是 varchar，但查询值未加引号，可能触发隐式类型转换导致索引不可用。

最小修改：改为 phone = '13800138000'，并用 EXPLAIN 验证 key=idx_phone、type 至少达到 ref。
