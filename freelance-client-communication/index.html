已在真实项目、工作或长期场景中反复使用，效果稳定，可以推荐别人直接参考。

场景

当你正在接外包、谈需求、报价、排期、处理临时加需求、催进度、压价、验收、售后或争议沟通时使用。它帮助你以专业、克制、可执行的方式回应甲方，而不是讨好、争吵或无限让步。

为什么这个 Prompt 存在

这个 Prompt 来自外包沟通里反复出现的现实问题：很多风险不是技术实现，而是边界、工期、价格、验收和售后没有说清楚。直接答应或直接拒绝，都会让后续沟通更难。

它解决什么问题

它解决的是甲方沟通中的边界判断：临时加需求算不算范围外，是否影响工期，是否需要报价，如何既不讨好也不激化矛盾。

我如何使用它

我会在收到甲方新要求、催进度、压价、验收争议或售后请求时使用它。先把聊天原话、已约定范围、报价、工期和自己的顾虑发进去，再让它判断阶段和回应策略。

为什么它被保留下来

它留下来，是因为它能把情绪化沟通拉回事实、范围和下一步。外包沟通最容易在口头承诺和临时让步里失控，这个 Prompt 能提醒我先确认边界，再给对方可发送的话术。

为什么变成今天这样

最早它主要帮我写回复。后来发现“怎么回复”通常不是第一个问题。更重要的是先判断：这是不是范围外需求，会不会影响工期，要不要重新报价，有没有验收和售后风险。 所以它后来从回复生成器变成了边界判断工具。先判断范围、工期、价格和风险，再给可发送的话术。

什么时候不要使用

不要把它当法律建议。不要在合同、付款、责任归属不清时让它替你下结论。不要用它生成攻击性话术。它适合沟通判断，不适合替代正式合同或法律处理。

- 甲方催进度、压价、反复改需求、要求售后或要求免费修改的情况
- 我希望表达的底线、可让步范围和不想激化矛盾的顾虑

# 判断维度

你必须覆盖以下判断：

1. 甲方诉求性质：正常需求、范围外需求、模糊需求、风险需求、压价需求、情绪化表达或争议沟通。
2. 范围判断：是否属于已约定范围，是否需要拆成新需求，是否需要重新确认边界。
3. 工期判断：是否影响原计划，是否需要延后，是否需要拆分交付。
4. 价格判断：是否应收费，是否可以免费处理，是否应按变更报价。
5. 风险识别：需求不清、口头承诺、验收标准缺失、无限修改、售后边界不清、情绪升级或付款风险。
6. 沟通策略：应该确认、解释、拒绝、让步、报价、拆分、延期、降温还是升级为书面确认。

# 专项处理规则

## 临时加需求

先判断是否超出原范围。不要直接答应。优先说明影响范围、工期和费用，再给可选方案。

## 催进度

先说明当前进度、已完成内容、剩余工作和下一个交付节点。不要空泛承诺“马上好”。

## 压价

先确认预算限制，再说明工作量、交付质量和风险边界。不要情绪化反击，也不要无条件降价。

## 验收与售后

必须区分 Bug 修复、范围内调整、范围外新增、长期维护和免费支持边界。

## 争议 / 扯皮降温

先降低对抗，回到事实、记录、范围、验收标准和下一步确认。避免讽刺、指责和情绪化表达。
