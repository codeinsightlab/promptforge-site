Bug定位 Prompt

场景

根据异常现象、复现步骤、日志和 Java 代码片段，定位最可能出错的位置并给出最小修复建议。

Prompt

复制

{{稳定或偶发复现步骤}}

{{日志/堆栈}}

示例输入

复制
异常现象：
用户未登录访问订单详情时返回 500，日志出现 NullPointerException。

复现步骤：
1. 清空 Cookie 和 Authorization
2. 请求 GET /api/orders/90001/detail
3. 偶发返回 500，未返回 401

代码片段：
public OrderDetail getDetail(Long orderId) {
    User user = UserContext.getCurrentUser();
    Order order = orderMapper.selectById(orderId);
    if (!order.getUserId().equals(user.getId())) {
        throw new BizException("无权限");
    }
    return buildDetail(order);
}

相关日志或堆栈：
java.lang.NullPointerException
at com.example.order.OrderService.getDetail(OrderService.java:42)

运行环境：
Spring Boot 2.7，JDK 17，订单服务通过网关转发，部分接口未配置登录拦截。

示例输出

复制
1. 最可能出错的位置（概率高）
   - user.getId()

2. 可疑逻辑
   - 未登录时 UserContext.getCurrentUser() 返回 null
   - 代码没有先判断 user 是否为空

3. 判断依据
   - 堆栈指向 OrderService.getDetail 第 42 行
   - 复现步骤明确未携带登录态

4. 最小修复建议
   - 在网关或拦截器中统一拦截订单详情接口
   - 方法内增加 user == null 判断并返回 401 或业务未登录错误

5. 建议补充测试用例
   - 未登录访问订单详情应返回 401
   - 登录用户访问他人订单应返回无权限
   - orderId 不存在时应返回明确业务错误
