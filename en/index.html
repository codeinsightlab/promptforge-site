Common Engineering Scenarios

When the problem type is already clear, enter the matching workflow directly.

Online Issue Troubleshooting Narrow production issues from impact, evidence, logs, API responses, and recent changes. Start troubleshooting → Performance and Stability Bottleneck Analysis Break down latency, timeouts, tail spikes, dependencies, resources, and stability risks. Analyze bottlenecks → SQL and Database Optimization Optimize slow SQL, invalid indexes, deadlocks, schema design, transactions, and data-volume risks. Optimize database → Cache and Redis Stability Review Review keys, TTL, consistency, hot keys, big keys, penetration, breakdown, and avalanche risks. Review cache → Pre-release API Review Review idempotency, permissions, security, error handling, compatibility, and regression risks before release. Review API → Distributed System Design Review Review architecture boundaries, consistency, transactions, MQ reliability, compensation, and reconciliation. Review design → Release and Monitoring Design Design rollout plans, rollback conditions, metrics, alert thresholds, and incident response actions. Design release →
