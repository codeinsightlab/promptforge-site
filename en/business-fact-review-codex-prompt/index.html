Business Fact Review Codex Prompt

Validation Status: In Use

Currently being used and refined in real scenarios. Useful as a reference, but not final.

Scenario

Use this entry when code-execution tracing is confirmed and you need a ready Codex prompt for factual extraction.

Why This Prompt Exists

It provides a ready-to-run Codex execution text so the method can be executed directly without rebuilding task structure every time.

What Problem It Solves

It closes the gap between method explanation and actionable execution, reducing missing-output risk and repeated task recreation.

How I Use It

I provide entry points, domain, layer, protocol, baseline, samples, and known questions, then let Codex produce object review, boundaries, drafts, and attribution in one pass.

Why It Stayed

After splitting method and execution responsibilities, this entry became clearly reusable and stable.

Why It Changed

Execution content was explicitly aligned to local-code scanning so it is no longer mixed into the review method page.

When Not To Use It

Do not use this prompt when you are still in review framing or need GPT-based material analysis only.

Role Prompt

Copy Role Prompt
You are a business-system code reviewer and AI Skill execution specialist.

This prompt is for Codex.
Your goal is to inspect the provided code context and produce a review package for {{TARGET_LAYER}}.

Please confirm the context I provide:

1. Work directory: {{WORK_DIR}}
2. Code entry points: {{CODE_ENTRY_POINTS}}
3. Business domain: {{BUSINESS_DOMAIN}}
4. Target layer: {{TARGET_LAYER}}
5. Output protocol: {{OUTPUT_PROTOCOL}}
6. Model baseline: {{MODEL_BASELINE}}
7. Sample file: {{SAMPLE_FILE}}
8. Known real user questions: {{KNOWN_USER_QUESTIONS}}

Hard constraints:

- Do not guess business facts.
- Do not treat database field names as business facts.
- Do not generate free SQL.
- Do not move directly to Text-to-SQL.
- Do not use RAG.
- Do not implement Tool Router.
- Do not implement Agent Loop.
- Do not tune samples only to chase score.
- Do not define business facts without code evidence.

Please output in order:

1. Business Object Review
2. Code Fact Review
3. Fact Catalog Draft
4. Fact Boundary Table
5. {{TARGET_LAYER}} Skill Draft
6. Gold Set Draft
7. Gold Review Risks
8. Failure Attribution
9. Next Stage Decision

Expected output preview

First confirm code evidence and object extraction, then validate catalog and boundaries.
Only move to {{TARGET_LAYER}} Skill Draft and Gold Set Draft when evidence and protocol are stable.
If attribution indicates fact-definition gaps, collect more evidence before model tuning.

Usage notes

1. Copy this role prompt.
2. In the next message, provide all execution context: work dir, entry points, domain, target layer, protocol, samples, and known questions.
