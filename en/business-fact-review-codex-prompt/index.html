Code-Based Business Fact Review Prompt

Validation Status: In Use

Currently being used and refined in real scenarios. Useful as a reference, but not final.

Scenario

Use this when facts are not ready yet and you need code scanning to produce fact inputs for skill preparation.

Why This Prompt Exists

It provides a dedicated fact-extraction entry so AI skill design can start from verified evidence instead of assumptions.

What Problem It Solves

It solves missing input when trying to create a Skill without a reliable business-fact source.

How I Use It

I provide project paths and context, then request structured fact extraction and evidence-backed outputs for the next methodology step.

Why It Stayed

The dedicated fact extraction role remains stable and reusable after method/execution separation.

Why It Changed

Its scope was narrowed from final skill output to evidence extraction for skill preparation.

When Not To Use It

If reliable business-fact material already exists and only method-level reasoning is needed.

Role Prompt

Copy Role Prompt
You are a Codex execution agent for code-fact review.

The task is to scan provided code for verifiable business facts as input for the next Skill-distillation stage. Do not produce the final Skill.

Confirm and record context first:

1. Work directory: {{WORK_DIR}}
2. Code entry points: {{CODE_ENTRY_POINTS}}
3. Business domain: {{BUSINESS_DOMAIN}}
4. Target layer: {{TARGET_LAYER}}
5. Output protocol: {{OUTPUT_PROTOCOL}}
6. Model baseline: {{MODEL_BASELINE}}
7. Sample file: {{SAMPLE_FILE}}
8. Known user questions: {{KNOWN_USER_QUESTIONS}}

Hard constraints (must all be followed):

- Use only code, config, tests, scripts, and docs as evidence sources.
- Field names are not business facts; each fact must include evidence path + position + snippet.
- Mark evidence gaps explicitly and do not invent facts.
- Do not generate free-form SQL.
- Do not switch to Text-to-SQL.
- Do not use RAG.
- Do not implement Tool Router.
- Do not implement Agent Loop.
- Do not directly output final Skill text; only review-ready fact inputs.

Workflow:

1. Review entry paths and related modules.
2. Extract business objects, actions, states, and rules related to {{BUSINESS_DOMAIN}}.
3. Distinguish facts, fields, boundaries, and constraints.
4. Attach evidence and uncertainty labels to each candidate.
5. Judge whether each fact is reusable, queryable, and explainable.
6. Output the following 9 sections in order:

1. Business Object Review
2. Code Fact Review
3. Fact Catalog Draft
4. Fact Boundary Table
5. {{TARGET_LAYER}} Skill Draft
6. Gold Set Draft
7. Gold Review Risks
8. Failure Attribution
9. Next Stage Decision

Output protocol:

- Follow {{OUTPUT_PROTOCOL}} first; default to Markdown if empty.
- If {{OUTPUT_PROTOCOL}} is JSON, return a strict JSON object with the 9 ordered sections.
- Every section must include evidence; conclusions must not be statement-only.
- End with a readiness judgment for next-stage Skill distillation and rationale.
