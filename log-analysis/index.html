日志分析 Prompt

场景

根据日志片段、时间范围、系统模块和用户操作，提取异常链路、关键错误点与排查方向。

Prompt

复制
你是一个 Java 后端日志分析专家，请根据以下信息分析日志：

【输入信息】
时间范围：
{{日志发生时间段}}

系统模块：
{{服务名 / 接口路径 / TraceId / 用户操作}}

日志片段：
{{完整日志片段，包含 ERROR/WARN/堆栈/TraceId}}

业务背景：
{{Java 后端业务流程、依赖服务、最近变更}}

【输出要求】
1. 关键错误点
2. 异常链路
3. 可能根因
4. 可能原因
5. 需要优先关注的日志行
6. 后续排查建议

【约束】
- 按日志时间顺序还原链路
- 每个根因必须给出日志依据
- 区分直接错误和上游诱因
- 信息不足时说明需要补充哪些 Trace、指标或配置

示例输入

复制
时间范围：
2026-04-28 10:12:00 ~ 10:13:00

系统模块：
order-service，POST /api/orders，用户提交订单，traceId=8f21a

日志片段：
2026-04-28 10:12:01 INFO OrderController - create order request, userId=3821, traceId=8f21a
2026-04-28 10:12:01 WARN PayClient - create payment timeout, orderId=90001, traceId=8f21a
java.net.SocketTimeoutException: Read timed out
at com.example.pay.PayClient.create(PayClient.java:57)
2026-04-28 10:12:02 WARN RetryTemplate - retry payment create, times=1, traceId=8f21a
2026-04-28 10:12:05 ERROR OrderController - create order failed, orderId=90001, traceId=8f21a
com.example.common.RemoteCallException: pay-service timeout

业务背景：
订单创建时同步调用 pay-service 创建支付单，支付单创建失败时订单返回失败。

示例输出

复制
1. 关键错误点
   - PayClient.create 调用 pay-service 发生 Read timed out
   - OrderController 最终抛出 RemoteCallException

2. 异常链路
   - 用户提交订单 -> order-service 创建订单 -> PayClient 调用 pay-service -> 超时重试 -> 订单创建失败

3. 可能根因
   - pay-service 响应慢或不可用
   - order-service 读超时时间偏短
   - 重试放大了下游压力

4. 需要优先关注的日志行
   - PayClient timeout 行
   - RetryTemplate retry 行
   - OrderController create order failed 行

5. 后续排查建议
   - 按 traceId 查询 pay-service 同时间日志
   - 查看 pay-service P95/P99、错误率和线程池指标
   - 核对 PayClient 超时、重试次数和熔断配置

使用说明

粘贴同一时间窗口内的完整日志和 TraceId，优先用于接口失败、远程调用异常和线上告警分析。
