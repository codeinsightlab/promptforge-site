日志分析 Prompt

场景

输入日志，提取异常模式与关键错误点

Prompt

复制
你是一个 Java 后端日志分析专家，请分析以下日志并提取异常模式与关键错误点：

日志内容：
{{日志内容}}

业务背景：
{{业务背景}}

请输出：
1. 关键错误点
2. 异常模式
3. 影响范围判断
4. 可能原因
5. 建议排查顺序

示例输入

复制
日志内容：
2026-04-28 10:12:01 ERROR OrderController - create order failed, userId=3821
java.net.SocketTimeoutException: Read timed out
at com.example.pay.PayClient.create(PayClient.java:57)
2026-04-28 10:12:02 WARN RetryTemplate - retry payment create, times=1
2026-04-28 10:12:05 ERROR OrderController - create order failed, userId=3822
java.net.SocketTimeoutException: Read timed out

业务背景：
订单创建时会同步调用支付系统创建支付单。

示例输出

复制
1. 关键错误点
   - PayClient.create 调用支付系统读超时

2. 异常模式
   - 多个用户在相近时间出现相同 SocketTimeoutException
   - 失败集中在订单创建链路

3. 影响范围判断
   - 影响依赖支付创建的订单提交请求

4. 可能原因
   - 支付系统响应慢
   - 网络连接不稳定
   - 本服务超时时间设置过短
