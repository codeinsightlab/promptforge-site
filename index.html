精选工作流

先选最接近的问题入口，让 GPT 收敛事实、风险和下一步动作；需要查代码时，再生成可交给 Codex 的最小任务。

线上问题排查 线上异常、接口失败、日志报错或表现不一致时，先收敛影响范围、证据链和最可能根因。 启动排查 性能与稳定性瓶颈分析 接口变慢、超时、P95/P99 抖动或服务连锁异常时，拆解调用链、依赖、资源和配置瓶颈。 分析瓶颈 SQL 与数据库优化 围绕 SQL、EXPLAIN、索引、表结构和数据量判断最小优化路径。 优化查询 缓存与 Redis 稳定性评审 评审 key、TTL、一致性、热点、大 Key、穿透、击穿和雪崩风险。 评审缓存 接口上线前评审 上线前检查接口幂等性、权限、安全、异常处理、兼容性和回归风险。 评审接口 分布式方案评审 评审服务边界、数据归属、调用链路、一致性、可用性和演进风险。 评审方案 发布与监控设计 围绕发布节奏、观测指标、告警阈值、异常升级和回滚条件设计上线保护。 设计保护 Tech Lead 技术判断 判断方案是否合理、是否过度设计，以及是否存在数据、状态或并发风险。 做技术判断 Codex + GPT 开发协作流 冻结需求边界，生成 Codex 执行任务，再把执行证据交回 GPT 审查。 组织协作

旧的细分排查入口已收口到主工作流中。先从高层工作流进入，再根据证据决定是否切换到更具体的数据库、缓存、架构或发布类页面。
